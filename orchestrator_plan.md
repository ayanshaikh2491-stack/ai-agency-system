# Website Builder Orchestrator Plan

## Overview
The goal is to orchestrate a set of specialized sub‑agents that will collaboratively build a complete website. Each sub‑agent has a clear responsibility and will work sequentially according to the defined workflow. The orchestrator will coordinate the hand‑off between agents, ensure that each deliverable meets the required specifications, and track progress using a checklist.

## Sub‑Agents and Responsibilities
1. **Planner Agent** – Gathers requirements, defines site structure, selects tech stack, and produces a detailed implementation plan.
2. **UI/UX Agent** – Designs the visual layout, component hierarchy, and responsiveness guidelines based on modern UI principles.
3. **Frontend Developer Agent** – Implements the UI using the chosen framework (e.g., React with Tailwind) following the UI/UX specifications.
4. **Backend Developer Agent** – Builds the server‑side APIs, database schema, authentication, and any business logic.
5. **Integration Agent** – Connects the frontend to the backend, resolves API mismatches, and ensures data flow works correctly.
6. **Testing Agent** – Executes functional, integration, and performance tests, logs bugs, and verifies edge‑case handling.
7. **Deployment Agent** – Deploys the frontend (Vercel/Netlify) and backend (Railway/Supabase), validates the live URLs, and provides final deployment links.

## Workflow
The orchestrator will move through the following steps, only proceeding when the previous step is marked complete:
1. **Planner Agent** creates a full plan.
2. **UI/UX Agent** designs the layout.
3. **Frontend Developer Agent** builds the UI.
4. **Backend Developer Agent** builds the API.
5. **Integration Agent** connects everything.
6. **Testing Agent** validates the product.
7. **Deployment Agent** launches the site.

The orchestrator will now invoke the **UI/UX Agent** to refine the design based on the implementation plan and existing UI/UX design. Following that, the Frontend Developer Agent will build/enhance the UI, then the Backend Developer Agent will create the server-side logic, and so on through the workflow.
